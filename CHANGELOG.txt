Major — complete UI/UX redesign.  No behavioural regressions; all 371 tests pass.

  Design system
  · Introduced CSS custom property palette (--bg, --surface, --accent,
    --green, --amber, --red, --purple, --teal, --pink); all colours now
    driven from one canonical source
  · Adopted DM Sans as the extension-wide typeface; replaces browser default
  · Consistent design tokens for border-radius, spacing, and shadow
  · Custom thin scrollbars throughout

  Popup — main panel  (popup.html, style.css)
  · Sticky header with MDE logo and FAQ / Facebook link buttons
  · Status bar replaces plain msgTextpop paragraph; includes:
    – Collapsible help text with more / less toggle (2-line clamp when
      collapsed, full text when expanded)
    – Dismiss button (x) to hide the bar entirely
    – State persisted across popup open/close via chrome.storage.local
  · All checkboxes replaced with custom animated toggle switches
  · Each section given a section-label heading and consistent padding
  · Toggle rows use flex layout with action buttons flush right
  · Sound Settings:
    – Table uses fixed column widths; play and reset buttons no longer
      clipped at the right edge
    – Configure button correctly toggles panel open and closed; label
      switches between "Configure" and "Close"
    – Cancel and Save both restore the button label to "Configure"
  · Quick Links rendered as a 2-column grid with a collapse / expand
    chevron; collapsed state persisted in chrome.storage.local
  · Text Messaging Setup header styled as an amber "DISABLED" banner;
    no non-ASCII characters used anywhere in CSS or HTML
  · All non-ASCII characters in popup.html replaced with HTML entities

  Productivity Tracker panel  (popup.html, style.css, popup.js)
  · Sticky header bar with panel title and close button
  · "This Data is for informational purposes only." rendered as a blue
    info callout with a left-border accent bar
  · Totals table: right-aligned time values, dimmed secondary labels,
    rounded card container
  · Consolidated/Device and Invoice controls restructured into two clean
    labelled rows (was a flat wrapped flex block)
  · Table controls split into a top row (header + filter dropdown) and
    an AET-range radio row
  · Timer Settings box redesigned with icon + title + hint layout for
    each of the four settings (Task Timer, Timer display, Keep records,
    Alert for incomplete tasks); backup options dropdown moved below the
    number input to prevent right-edge overflow
  · Data management section (Rebuild, Restore, Delete) given section
    label and consistent button styling
  · Row status colours updated to design-token values: amber (released),
    teal (incomplete), pink (in-progress), purple (prior pay periods)

  Tooltip copy  (popup.js)
  · All 26 help[] entries rewritten for clarity, accuracy, and consistent
    voice; removed raw task-page shortcut dumps from popup-context entries

  Bug fixes (identified during UI/UX work)
  · popup.js — liClick ReferenceError: .click() call sites still referenced
    the old name after the 1.0.0 rename to popup_liClick; updated both
    call sites
  · popup.js — PLAYTHIS message handler returned bare return with no
    sendResponse call, leaving the message port unclosed
  · popup.js — hoverCallback fired on child element re-entries, resetting
    save_message to defaultHelp mid-hover and flashing "No help found";
    now guarded with a help-active class to capture save_message only
    once per hover sequence
  · popup.js — FAQ panel rendered as an inline block, pushing all content
    below it down when opened; replaced with a position:absolute overlay
  · background.js — myListener delegated to real_myListener without
    returning true at the top level; Chrome closed the message port before
    any async response could be sent, producing "Uncaught runtime.lastError:
    The message port closed before a response was received" on every
    message exchange
  · style.css / popup.html — body width hardcoded at 340px with
    overflow-x:hidden, clipping the Productivity Tracker panel; body now
    gains a tracker-open class when the tracker is shown, expanding to
    580px minimum width
  · style.css / popup.html — defaultHelp version string displayed as
    "MDE v.1.0(05/08/2026)" (no space before parenthesis); corrected

  utils.js
  · Fixed: setURL(suffix, international) — international branch correctly

    unconditionally reset prefix = USRoot, making the international flag
    a no-op; international users' invoice URLs always resolved to

  · Fixed: NRTLogCleanup(logData) returned newlogData only when records
    were pruned; on days with no prunable records the function fell
    through and returned undefined, causing a TypeError at the call site
    (nrtRecs.push()) and silently preventing the NRT log storage write
    Corrected to unconditionally return newlogData

    convert2Pacific across all 8 references (1 definition, 7 call sites)

  phrases.js
  · Fixed: variable z declared as let z inside the for loop initialiser
    but read outside the loop body on the next line; block-scoping caused
    a ReferenceError in strict mode and the Node vm sandbox
    Corrected by hoisting: let z = 0 before the loop

Major — project recovery and re-release under new ownership.

  The extension was inherited in an unmaintained state after the original
  developer became unreachable.  The codebase was functional but carried
  significant technical debt accumulated over its prior development history:
  the original branding was intact but tied to infrastructure no longer in
  use, and the code had never been subjected to a systematic quality review.

  This release represents the minimum necessary to establish a clean,
  documented, and correctly branded foundation to build on.

  Ownership and identity
  · Extension renamed from LFH to MDE across all user-visible surfaces:
    popup title, status messages, storage keys, and support contact
  · All source files renamed to kebab-case for consistency
  · Support contact updated to mdipros48@gmail.com
  · Facebook community page established for user communication

  Infrastructure cleanup
  · SMS backend removed — the third-party service it depended on is no
    longer available; the feature is preserved in the UI as disabled,
    pending a replacement implementation
  · Orphaned files with no remaining references removed from the package
  · All storage keys migrated to the new namespace without data loss

  Code modernisation  (29 JS files, ~7,000 lines affected)
  · Removed approximately 370 console.log statements that had been left
    in place from development and were exposing internal state in
    production builds
  · Converted all var declarations to let across every JS file;
    var's function-scoping had been the source of several subtle bugs
    and makes the code significantly harder to reason about
  · Identified and corrected 35 implicit global variable assignments
    across 12 files — variables written without a declaration silently
    become properties on the global window object, creating potential
    for unexpected cross-script interference in the extension's
    multi-context execution environment
  · Removed approximately 700 lines of dead code: unreachable branches,
    commented-out blocks, and functions with no remaining call sites
  · Resolved all same-name function conflicts; each function now has a
    unique, unambiguous identifier across the entire codebase
  · Added JSDoc documentation to all 390 functions, establishing a
    consistent reference for anyone working on the codebase going forward
