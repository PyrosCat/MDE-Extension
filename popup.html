This extension requests these permissions for these reasons:

The rest of this FAQ is specifically related to the tracker timer.
How do I start using the timer?
-- install the extension
-- open the popup, select the check box next to "Productivity Tracker"
-- Work a task - and submit it
How do I get an in task timer?
-- Open the popup and hit the pink "details" button next to the productivity Tracker
-- A new popup is displayed that has info about what you have worked.
-- Below the table with tasks (green lines can be clicked to see detail for the day) there is a drop down that says no in-task timer - pull that down and select a time to be warned before the AET is up.
-- Next task you work, you will see the count down in the tab header of the task
What happens when you hit the back button or navigate away from the task while working?
A task record is generated and saved. When you go back to the task, there will now be two records in the detail display for this task. When you submit or release the task, the records will be consolidated.
How is time on released tasks handled?
-- The time you worked on the task will be counted for the task
-- The task will show up as orange in the detail display
-- The AET will be set to the time worked so the productivity calculation is remains correct
-- In the AET total for the day and week, the task AET will not be counted.
How do I pause in the middle of a task?
The MDE timer for a task starts when you open a task (select it from raterhub) and stops when you submit, release, or close the task tab without doing either. The only way to pause the MDE timer mid-task is to close the task tab. When you come back to continue the task, and complete it (or release it) the time spent working will be consolidated into one record.

When a task is acquired from the RH index, and you are working it with an open tab, the task status will be in progress.

After aquiring a task, if the tab with the task is closed via you hitting the cancel button, or closing the tab without submitting or releasing, the task will be marked as incomplete. Once you reaquire the task (with the Continue button from the RH index), a new record will be created for it, you will see that new record in pink as an in progress task. MDE will now show two records for the task, one will be blue, marked as incomplete, and the other will be pink, marked as in progress. After finally submitting or releasing the task, the two records will be merged and you will now see the status changed to either submitted (record will have no background color) or released (orange).

If you have an incomplete task, and you think you actually did submit it, you can change the status to be submitted by clicking on the work time field for its entry. After clicking this field, to the right of the record, you will see buttons that can be used to modify this record. One of them is "2SUB" - this is the button to mark the task as submitted. You can also delete the record with the "Delete Row" button.

If you have released a task, it will be marked as released. When you release a task the AET for the task is set to the time you spent working on it before releasing. This is why the AET you see for the day or week may not match the true AET. This is done for two reasons; to keep the productivity rate accurate and your work time field will reflect your time worked on it, even when you have released it.

There have been times that for some reason MDE will fail to record a task correctly. Reasons could be an undiscovered bug in its code, or some weird thing happening on your computer. For instance, if you start getting a lot of incomplete tasks, or MDE doesn't record the AET or work time correctly for multiple tasks, please let me know, this is not a normal occurance, and at that point your best option is to rely on another method for time keeping until the reason is found.
